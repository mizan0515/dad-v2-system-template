# 01. 시스템 감사

## 목적

현재 저장소에서 DAD v2 시스템과 관련 문서/도구/세션 아티팩트가 실제로 맞물려 동작하는지 점검한다.

## 기본 감사 항목

1. `AGENTS.md`, `CLAUDE.md`, `DIALOGUE-PROTOCOL.md` 간 규칙 충돌 유무
2. `.claude/commands/`, `.agents/skills/`, `.prompts/`의 경로와 절차가 루트 계약 문서와 일치하는지
3. `Document/dialogue/` 세션 구조와 validator 기대치가 일치하는지
4. `tools/Validate-DadPacket.ps1`, `tools/Validate-Documents.ps1`가 실제 산출물 기준으로 맞게 동작하는지
5. 문서 drift가 남아 있으면 같은 턴에서 수정되었는지, 아니면 명시적 후속 작업으로 기록되었는지

## 출력 형식

- `PASS`: 문제 없음, 파일/라인 근거 명시
- `FAIL`: 문제 발견, 현재값/기대값/수정 diff 명시
- `WARN`: 즉시 위반은 아니지만 후속 보강 권장

## 원칙

- 실파일 우선
- validator 결과를 맹신하지 말고 validator 자체의 결함도 감사 대상에 포함
- 이미 고친 문제를 재보고하지 말고, 남은 gap만 보고
