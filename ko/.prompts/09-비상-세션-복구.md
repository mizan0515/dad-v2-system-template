# 09. 비상 세션 복구

## 목적

정상적인 세션 재개가 불가능할 때, DAD v2 세션을 강제로 닫거나 `state.json`을 수동 복구하고 운영 위험을 최소화한다.

## 언제 사용하나

- `Document/dialogue/state.json` 또는 `sessions/{session-id}/state.json`이 깨졌을 때
- turn packet은 남아 있는데 state가 최신 턴과 어긋날 때
- `tools/Validate-DadPacket.ps1` 자체가 실패하거나, validator 결함 때문에 정상 재개가 막힐 때
- force-close 후 새 세션으로 넘어가야 할 때

## 비상 절차

1. 현재 루트 `state.json`, 세션별 `state.json`, 마지막 `turn-{N}.yaml`을 모두 읽는다.
2. 손상 범위를 구분한다.
   - packet은 정상인데 state만 틀린지
   - packet도 부분 손상인지
   - validator가 잘못된 것인지
3. packet이 더 신뢰 가능하면 packet 기준으로 state를 복구한다.
4. packet도 신뢰할 수 없으면 세션을 `abandoned` 또는 `superseded`로 force-close 한다.
5. force-close 시 아래를 동시에 맞춘다.
   - 루트 `Document/dialogue/state.json`
   - `Document/dialogue/sessions/{session-id}/state.json`
   - `summary.md`
   - 닫힌 세션용 named summary
6. 복구 후 `tools/Validate-DadPacket.ps1 -Root . -AllSessions`를 다시 시도한다.
7. validator가 여전히 고장 나면 validator 결함을 명시하고, 우회 근거를 session summary 또는 handoff에 남긴다.

## 수동 state 복구 체크리스트

최소 필드:

- `protocol_version: "dad-v2"`
- `session_id`
- `session_status`
- `relay_mode: "user-bridged"`
- `mode`
- `scope`
- `current_turn`
- `max_turns`
- `last_agent`
- `contract_status`
- `packets`

복구 규칙:

- `current_turn`은 마지막 `turn-{N}.yaml`의 `turn`과 같아야 한다.
- `last_agent`는 마지막 packet의 `from`과 같아야 한다.
- `packets`는 실제 존재하는 packet 상대경로 목록과 같아야 한다.
- 닫힌 세션이면 `closed_reason`이 필요하다.
- `session_status: superseded`면 `superseded_by`가 필요하다.

## Force-close 기준

- packet과 state가 모두 부분 손상되어 복구 근거가 불충분할 때
- validator를 고쳐도 기존 세션 산출물이 신뢰 불가일 때
- 사용자에게 새 세션 재시작이 더 안전할 때

권장 상태값:

- 복구 포기: `abandoned`
- 새 세션으로 교체: `superseded`

## Validator 장애 시 우회 규칙

1. validator 오류가 세션 데이터가 아니라 validator 코드 결함인지 먼저 입증한다.
2. 입증 근거를 파일/라인 단위로 남긴다.
3. 가능하면 같은 턴에 validator를 수정한다.
4. 같은 턴에 수정할 수 없으면 다음 handoff의 첫 작업으로 validator 수정과 재검증을 적는다.
5. 우회 상태로는 `suggest_done: true`를 기록하지 않는다.

## 금지

- packet을 읽지 않고 state만 수동 편집하기
- force-close 후 summary 없이 종료 처리하기
- validator 결함을 확인하지 않고 임의로 검증을 생략하기
- 근거 없이 `converged`로 닫기
