# 06. 수렴 종료 / PR 정리

## 목적

세션이 거의 끝났을 때 수렴 판정, summary 산출물, 검증, 브랜치/PR 정리를 빠뜨리지 않게 한다.

## 언제 사용하나

1. 양쪽 에이전트가 주요 체크포인트를 모두 PASS로 보고할 때
2. `handoff.suggest_done: true`를 기록하려고 할 때
3. 세션을 `converged`로 닫고 커밋, push, PR, merge 후속 절차를 진행할 때

## 종료 체크리스트

1. 마지막 turn packet의 Contract 체크포인트가 모두 PASS인지 확인한다.
2. `handoff.suggest_done`가 true면 `handoff.done_reason`이 구체적으로 채워져 있는지 확인한다.
3. `Document/dialogue/state.json`과 세션별 `state.json`이 `converged` 상태를 정확히 반영하는지 확인한다.
4. `summary.md`와 닫힌 세션의 named summary가 모두 존재하는지 확인한다.
5. 최소 검증을 다시 실행한다.
- `tools/Validate-Documents.ps1 -Root . -IncludeRootGuides -IncludeAgentDocs -Fix`
   - `tools/Validate-DadPacket.ps1 -Root . -AllSessions`
6. 브랜치 상태를 확인하고 불필요한 변경이 섞이지 않았는지 확인한다.
7. 커밋, push, PR, merge 절차를 루트 계약 문서 정책에 맞게 처리한다.

## 출력 형식

- `PASS`: 종료 요건 충족, 근거 파일/라인과 검증 명시
- `FAIL`: 빠진 종료 산출물 또는 절차 위반, 현재값/기대값/수정 diff 명시
- `WARN`: 지금은 종료 가능하지만 후속 운영 리스크가 남은 경우
