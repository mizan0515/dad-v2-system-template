# 05. 의견 차이 / 디베이트 정리

## 목적

Codex와 Claude Code가 같은 checkpoint에 대해 PASS/FAIL이 갈릴 때, 감상평이 아니라 근거 기반으로 정리하고 수렴 또는 escalate한다.

## 언제 사용하나

- 한쪽은 PASS, 다른 쪽은 FAIL일 때
- 코드/문서/validator 해석이 충돌할 때
- 같은 checkpoint가 반복해서 막힐 때

## 수행 순서

1. 어느 checkpoint에서 갈렸는지 명시한다.
2. 각자 근거를 코드, 테스트, 로그, 문서로 분리해서 적는다.
3. 상대 근거 중 수용되는 부분과 반박할 부분을 나눈다.
4. 가능하면 합의된 부분만 PASS로 확정하고 남은 쟁점만 좁힌다.
5. 3라운드 안에 합의가 안 되면 사용자 escalate로 넘긴다.

## 품질 기준

- "좋아 보인다", "대체로 맞다" 같은 표현 금지
- line reference 또는 실행 근거 없는 판정 금지
- 부분 합의가 가능하면 전체 FAIL로 뭉개지 말 것
