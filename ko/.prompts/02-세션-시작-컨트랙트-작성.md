# 02. 세션 시작 / 컨트랙트 작성

## 목적

새 DAD v2 세션을 시작할 때 under-scope 없이 첫 Turn의 Contract와 실행 범위를 잡는다.

## 언제 사용하나

- 새 세션을 시작할 때
- 기존 세션을 supersede하고 새 세션으로 갈아탈 때
- 사용자의 요청이 짧거나 모호해서 곧바로 체크포인트로 내리기 어려울 때

## 수행 순서

1. `PROJECT-RULES.md`, `AGENTS.md` 또는 `CLAUDE.md`, `DIALOGUE-PROTOCOL.md`를 읽는다.
2. 라이브 파일 상태와 현재 브랜치를 확인한다.
3. 필요하면 `task_model`을 먼저 만든다.
4. `success_shape`, `major_risks`, `out_of_scope`를 기준으로 3~5개 체크포인트를 만든다.
5. Turn 1에서 가능한 첫 수직 슬라이스까지 같이 실행한다.
6. Turn Packet에 Contract 초안과 첫 실행 근거를 함께 남긴다.

## 체크포인트 품질 기준

- 구현 산출물 기준으로 판정 가능해야 한다.
- 검증 방법이 checkpoint에 같이 들어가야 한다.
- `out_of_scope`를 몰래 다시 끌어오면 안 된다.
- 프로젝트 문서 drift가 보이면 별도 checkpoint 또는 후속 작업으로 드러내야 한다.
