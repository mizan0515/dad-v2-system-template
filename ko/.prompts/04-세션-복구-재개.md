# 04. 세션 복구 / 재개

## 목적

중단된 DAD 세션을 잘못 이어붙이지 않고, 마지막 유효 상태에서 재개한다.

## 언제 사용하나

- 대화가 끊긴 뒤 다시 이어갈 때
- 다른 에이전트 결과를 늦게 받아 재개할 때
- state와 packet이 엇갈리는지 확인해야 할 때

## 수행 순서

1. `Document/dialogue/state.json`을 읽는다.
2. 현재 `session_id`, `session_status`, `current_turn`, `last_agent`를 확인한다.
3. 세션 디렉터리의 마지막 `turn-{N}.yaml`를 읽는다.
4. `tools/Validate-DadPacket.ps1 -Root . -AllSessions` 또는 대상 세션 검증을 실행한다.
5. `active`가 아니면 이어붙이지 말고 새 세션 필요 여부를 판단한다.
6. drift나 missing artifact가 있으면 먼저 복구하고, 그 다음 턴을 시작한다.
7. `state.json`이 깨졌거나 validator 자체가 고장 난 경우에는 일반 재개 대신 `.prompts/09-비상-세션-복구.md` 절차로 전환한다.

## 금지

- validator 실패 상태에서 다음 턴을 계속 쓰기
- superseded / abandoned 세션을 active처럼 이어붙이기
- state만 믿고 packet을 읽지 않기
