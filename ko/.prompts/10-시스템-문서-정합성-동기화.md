# 10. 시스템 문서 정합성 동기화

## 목적

작업 중 DAD 인프라나 시스템 규칙이 바뀌었을 때 코드/validator/명령/문서가 서로 drift되지 않게 한다.

## 시스템 문서 세트

- `AGENTS.md`
- `CLAUDE.md`
- `DIALOGUE-PROTOCOL.md`
- 루트 프로토콜이 얇게 유지되는 경우 `Document/DAD/` 상세 참조 문서
- `.claude/commands/`
- `.agents/skills/`
- `.prompts/`
- `Document/` 내 운영 가이드
- `tools/Register-CodexSkills.ps1`
- `tools/Set-CodexSkillNamespace.ps1`
- `tools/Validate-CodexSkillMetadata.ps1`
- `tools/Validate-DadPacket.ps1`
- `tools/Validate-Documents.ps1`

## 언제 사용하나

아래 중 하나라도 해당하면 이 프롬프트를 기본 참조로 포함한다.

1. session 경로, packet/state schema, summary naming 규칙이 바뀔 때
2. slash command, skill, prompt template, validator가 바뀔 때
3. 루트 계약 문서들 사이 충돌을 발견했을 때
4. 실 저장 구조와 문서 규칙이 어긋난 것을 발견했을 때
5. 작업 결과를 이해하려면 시스템 운영 문서도 같이 바뀌어야 할 때

## 실행 규칙

1. 실파일 기준으로 authoritative rule을 확인한다.
2. 바뀌어야 하는 시스템 파일을 목록화한다.
3. 같은 작업 안에서 가능한 한 모두 수정한다.
4. 수정 후 최소한 아래를 다시 돌린다.
   - `tools/Validate-Documents.ps1 -Root . -IncludeRootGuides -IncludeAgentDocs -Fix`
   - skill naming 규칙이 바뀌었다면 `tools/Set-CodexSkillNamespace.ps1 -Namespace "<repo-prefix>"`
   - `tools/Validate-CodexSkillMetadata.ps1 -Root .`
   - `tools/Validate-DadPacket.ps1 -Root . -AllSessions`
5. 같은 턴에 다 닫지 못하면 `handoff.next_task`의 첫 항목으로 시스템 문서 동기화를 명시한다.

## 금지

- 코드만 맞추고 시스템 문서는 암묵적으로 미루기
- validator가 틀렸는데 세션 파일만 우회 수정해서 통과시키기
- 루트 계약 문서와 하위 command/skill/prompt 충돌을 남긴 채 `PASS` 처리하기
