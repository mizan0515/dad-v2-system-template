# 03. 턴 종료 / 핸드오프 정리

## 목적

한 턴의 결과를 Turn Packet, state, handoff prompt, 남은 리스크까지 일관되게 마감한다.

## 언제 사용하나

- 각 턴 종료 직전
- self-iteration을 마치고 peer handoff를 만들 때

## 수행 순서

1. 이번 턴의 실제 변경과 검증 근거를 정리한다.
2. 각 checkpoint를 PASS / FAIL / FAIL-then-FIXED / FAIL-then-PASS로 평가한다.
3. `issues_found`, `fixes_applied`, `open_risks`를 분리한다.
4. `handoff.next_task`와 `handoff.context`를 다음 에이전트가 바로 이어받을 수 있게 쓴다.
5. 시스템 문서 drift가 남았으면 `handoff.next_task` 첫 항목으로 명시한다.
6. `suggest_done: true`를 쓰기 전 validator를 돌린다.
7. 상대용 프롬프트 6개 필수 요소와 mandatory tail을 확인한다.

## Done Gate 체크

- 최신 수정 이후 peer PASS가 있는가
- evidence가 재현 가능한가
- validator가 통과했는가
- 남은 리스크가 암묵적으로 숨겨져 있지 않은가
